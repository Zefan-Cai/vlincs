large component merges are dangerous false-merge candidates.  This follow-up
kept the same no-anchor candidate source but changed the rule: only tiny
source fragments were allowed to move into much larger target components.

candidate features only.  GT is used only by the canonical DS1 evaluator after
candidate assignment CSVs are produced.

## Interpretation

This is not a win, but it is a useful discriminator:

- broad component bridge best p0.5 IDF1: `0.648646`
- small-fragment bridge best p0.5 IDF1: `0.655486`
- current best p0.5 IDF1: `0.655911`

So the hypothesis is partly right: small fragments avoid most of the broad
false-merge damage.  The missing piece is impact and target correctness.  The
top row is only `0.000425` below best, which makes it a good hard negative /
near-positive for a side-effect critic, but not a production candidate.
